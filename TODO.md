# 待办事项清单

1. 拆解 `chat-skills` 规则体系  
把 `SKILL.md`、`input-rules.md`、`data-quantization-sop.md`、`algorithm-weight.md`、`output-rules.md`、`stage-model.md` 整理成程序可执行逻辑。

2. 设计分析流水线  
形成固定链路：`输入内容 -> 特征提取 -> 参数量化 -> 核心指标计算 -> 结果生成`

3. 设计阶段识别逻辑  
基于 `stage-model` 输出当前互动阶段，如初识、升温、舒适区、邀约窗口、冷淡/冲突等。

4. 设计核心指标体系  
首版至少实现并展示：
- `IVI` 意图真实度
- `SPE` 社交势能
- `EWS` 升温窗口

5. 设计结果输出结构  
结果页统一输出：
- 对方心理/态度倾向
- 当前互动阶段
- 核心指标结论
- 下一步聊天建议
- 可直接发送的话术参考

6. 设计参数面板方案  
作为唯一保留的可选项，评估是否展示 `Sp`、`Fback`、`Pface`、`Rlatency` 等细粒度参数，以及展示方式。

7. 实现输入模块  
支持两种输入：
- 上传单张长聊天截图
- 粘贴聊天文本

8. 实现图片预处理  
对长截图做基础预处理，保证后续识别与分析可用。

9. 实现 OCR 与结构化解析  
把截图提取为消息内容，并尽可能识别文本、消息顺序、时间特征、语气特征。

10. 实现文本直分析  
纯文本输入可直接进入分析，不依赖图片识别。

11. 实现分析引擎  
完成规则映射、参数量化、阶段判断、核心指标计算和策略生成。

12. 实现聊天建议生成  
给出 2～3 条建议，覆盖推进、观察、收缩等不同策略。

13. 实现回复文案生成  
输出几条自然、可直接参考的回复话术。

14. 实现结果展示页  
以网页应用形式展示完整分析结果，适合桌面端浏览和远程演示。

15. 实现基础历史记录  
保留最近分析结果，方便回看。

16. 加入免责声明与基础风控文案  
在页面中加入“仅供沟通参考，不构成心理诊断”等说明。

17. 完成云端部署方案  
准备生产环境配置、部署方式、公网访问方案。

18. 部署 Demo 到云服务器  
完成上线并确保可通过网址远程访问。
