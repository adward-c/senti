2. `[待执行]` 准备空服务器基础环境  
在服务器安装 Docker、Docker Compose 插件、Git、必要系统依赖和基础防火墙规则。确认 `docker compose version` 可用，服务器时间同步正常，项目部署目录和权限规划清晰。

3. `[待执行]` 将项目代码部署到服务器  
根据仓库可访问性选择 `git clone`、Deploy Key、GitHub token 或本地打包上传。部署目录不保存明文凭据；后续更新优先通过可重复的拉取 / 构建流程完成。

4. `[待执行]` 配置生产 `.env` 与最小权限  
在服务器项目目录创建 `.env`，填写数据库密码、Kimi Key、CORS 域名、邀请码、认证密钥、Grafana 管理员密码等生产配置。`.env` 不写入仓库，不输出到日志，设置最小文件权限，例如仅 root 可读写。

5. `[待执行]` 调整公网反向代理暴露面  
后端容器不直接暴露到公网，只通过 Nginx 访问；PostgreSQL 仅绑定本机或 Docker 内网；Prometheus、Grafana 仅允许通过 SSH 隧道或服务器本机访问。公网 Nginx 不暴露 `/metrics`。

### P0-B：M2 MVP 留存闭环必须完成

10. `[待执行]` 明确 M2 报告契约与指标口径  
确认阶段枚举、中文标签、置信度字段、IVI / SPE / EWS 指标定义、风险类型、风险等级、`blocked` 规则、行动卡字段和免责声明字段。该契约应作为后端响应、数据库保存、前端渲染和测试断言的共同依据。

11. `[待执行]` 设计并迁移 M2 数据模型  
新增或调整关系档案、分析与档案绑定、结构化消息、行动标签、反馈记录、支付状态等表结构。保证反馈必须关联用户和分析记录，若分析已绑定档案则同时关联档案。

12. `[待执行]` 升级后端分析记录保存格式  
保存确认后的结构化消息、稳定报告结构、关键证据、风险拦截结果、行动卡和低置信度标记。历史详情接口需要兼容旧记录，并能返回 M2 页面所需字段。

13. `[待执行]` 支持长聊天文本导入并解析为结构化消息  
用户可以粘贴长聊天记录；系统解析为消息列表，并尽量识别说话人、时间或分段。解析失败时仍要允许用户手动修正。

14. `[待执行]` 支持 1 至 9 张截图按顺序导入  
截图上传应保留用户选择顺序；OCR 结果需要合并为可编辑的结构化消息列表。需要处理 OCR 失败、部分图片失败、顺序异常和超出数量限制的提示。

15. `[待执行]` 增加分析前会话确认与编辑界面  
用户在提交分析前可以编辑消息文本，并修改说话人身份为“自己 / 对方”。只有确认后的结构化消息才能进入分析流程。

16. `[待执行]` 升级分析报告生成逻辑  
每次分析返回稳定一致的报告结构：阶段、态度摘要、IVI / SPE / EWS、至少三条证据、推荐回复、备选回复、低置信度标记、行动卡、风险字段和免责声明。

17. `[待执行]` 强化风险拦截与安全回复策略  
对黄暴相关信息和其他高风险场景设置风险等级、风险类型、风险说明和 `blocked` 标记。被拦截时不得生成不安全推荐回复，目标是风险拦截回复安全性达到 100%。

18. `[待执行]` 实现关系档案创建、列表与编辑  
档案字段包括昵称、渠道、当前阶段、当前风险等级、当前动作标签、距离上次分析之间的实际动作标签、创建时间和更新时间。用户输入字段与分析输入字段要区分来源。

19. `[待执行]` 实现分析保存与档案绑定流程  
一个分析结果可以不绑定，也可以精确绑定到一个档案。需要确定主流程入口：优先评估“先建档再分析且默认绑定”与“先分析再无感绑定”两种方案，并落地一种默认路径。

20. `[待执行]` 实现档案详情洞察页  
档案详情页展示最新阶段、最新动作卡、最新风险状态、阶段趋势、最近 5 次分析的 IVI / SPE / EWS 简化趋势，以及已绑定分析的时间线。

21. `[待执行]` 实现档案时间线记录  
时间线展示每次分析的摘要、关键证据、阶段、风险和反馈状态，支持进入对应分析详情。

22. `[待执行]` 支持从档案直接发起新分析  
档案详情页提供新分析入口，跳转后默认携带并绑定当前档案，减少用户重复选择成本。

23. `[待执行]` 支持用户反馈“准 / 不准”  
用户可以对每次分析标记“准确 / 不准确”，并可选填写原因说明。反馈数据用于后续优化规则、提示词和报告可信度。

24. `[待执行]` 增加基础支付系统  
实现 M2 所需的基础支付闭环，包括支付状态记录、订单或订阅记录、支付成功回调处理、权限校验和前端状态展示。若暂不接真实支付渠道，至少保留可替换的接口与数据模型。
